# ShufflePic v1.0 操作マニュアル

指定フォルダ直下の画像を 1〜2 枚並べてランダムにスライドショー表示する Windows 向けアプリです。
**原本は読み取り専用**（「削除」と「巨大画像の自動退避」を除く）で、いずれもフォルダ内のサブフォルダへ
**移動**する方式です（消去ではありません）。

> 前身 からの主な変更：Undo 廃止／管理画面（フォルダ・間隔・表示枚数）／設定とサイクルの**ディスク永続化**／
> 巨大画像の**自動退避**／デコードの**別スレッド化**。

---

## 1. 動作環境

- **OS**: Windows 10 1607 以降 / Windows 11
- **ビルド/実行**: Rust ツールチェーン（stable）
- **日本語フォント**: `meiryo` / 游ゴシック / MS ゴシック のいずれか（標準的な Windows なら自動検出）
- **対応画像形式**: `jpg` / `jpeg` / `png` / `gif` / `bmp` / `webp`（大文字小文字不問）
  - `gif` は静止画として表示（アニメーションしません）。EXIF 回転は反映しません。

---

## 2. 起動とフォルダ指定

```
cd D:\Library\programming\Rust\ShufflePic\v1.0
cargo run --release
```

> **必ず `--release`** で実行してください（デバッグビルドはデコードが遅い）。

- 前身 と異なり、**コンソールでのパス入力は廃止**しました。
- 起動時、前回の状態ファイルがあり、そのフォルダが今も存在すれば**前回の続きから再開**します（§6）。
- 状態が無い／フォルダが無い場合は、画面中央に「**設定からフォルダを選択してください**」と表示されます。
  上部バーの「**⚙ 設定**」からフォルダを選ぶと開始します。

## 3. 画面の見かた

```
┌───────────────────────────────────────────────────────┐
│ ⏸ 停止  ⚙ 設定 │ 表示済み:12  未表示:108  キャッシュ:3  サイクル:1 │ ← 上部バー
├───────────────────────────────────────────────────────┤
│                                                       │
│        ［ 左の画像 ］        ［ 右の画像 ］             │ ← 画像表示エリア
│                                                       │
└───────────────────────────────────────────────────────┘
```

### 上部バーの項目

| 表示 | 意味 |
| --- | --- |
| `⏸ 停止` / `▶ 再生` | 現在の再生状態（押せます） |
| `⚙ 設定` | 管理画面（フォルダ・間隔・表示枚数）を開閉 |
| `表示済み: N` | このサイクルで既に表示し終えた枚数（= 内部の `cursor`） |
| `未表示: N` | このサイクルでまだ表示していない枚数 |
| `キャッシュ: N` | メモリに載っている表示用テクスチャの枚数 |
| `サイクル: N` | 全画像を出し切って再シャッフルした回数 |

- 「表示済み」＋「未表示」＝総枚数 になります。
- 一時的なエラーは上部バーに**赤文字で数秒**表示され、自動で消えます（モーダルではありません）。

### 画像表示エリア

- 表示枚数の設定（1 または 2）に従って並べます。高さを揃え、領域に収まるよう縮小（アスペクト比維持）。
- 画像が無いときは「画像がありません（空フォルダ状態）」、読み込み中は「読み込み中...」と表示します。

## 4. 操作一覧

### 4.1 キーボード

| キー | 動作 |
| --- | --- |
| `Space` | 再生 / 停止の切り替え |
| `F11` | 全画面表示の切り替え |

> `Ctrl+Z`（Undo）は**廃止**しました。

### 4.2 マウス

| 操作 | 動作 |
| --- | --- |
| 画像を**右クリック** | メニュー（「再生／一時停止」「削除」「キャンセル」）を表示 |
| 画像エリアの**背景をダブルクリック** | 全画面表示の切り替え（画像の上ではなく余白） |
| 上部の `▶ 再生` / `⏸ 停止` | 再生 / 停止 |
| 上部の `⚙ 設定` | 管理画面の開閉 |

> 右クリックメニューや設定画面を開いている間は、**自動送りが一時停止**します（§5.2）。

## 5. 主な動作の仕様

### 5.1 スライドショー

- 切り替え間隔は **5〜60 秒**（管理画面で設定。既定 15 秒）。
- **1 サイクル内は重複なし**でランダム表示し、全部出し切ったら再シャッフルして繰り返します。
- 起動直後や切替直後は、必要な分からバックグラウンド（別スレッド）でデコードします。間に合わない間は
  「読み込み中...」と表示し、UI は固まりません。

### 5.2 メニュー／設定中の一時停止（残り時間から再開）

- 右クリックメニューまたは管理画面を開いている間は、自動送りのカウントが**凍結**します。
- 閉じると、**凍結した残り時間が経過した時点で**次へ送られます（例：残り 3 秒で開いたら、閉じてから 3 秒）。
- ただし、停止中にメニューの「再生」を選んだ場合は、**設定間隔を最初から計り直し**ます。

### 5.3 削除（重要）

- 画像を右クリック →「削除」で、その画像を **`（指定フォルダ）\delete` へ移動**します。
- **完全削除ではありません。** ファイルは `delete` に残ります。本当に消すなら後で手動で削除してください。
- 移動先に同名があっても**上書きしません**（`名前 (1).拡張子` … と別名退避）。
- **Undo はありません。** 戻したい場合は `delete` フォルダから手動で戻してください。
- メニューを開いたまま対象が表示から切り替わった場合は、誤削除防止のため**削除はキャンセル**されます。

### 5.4 巨大画像の自動退避（新）

- 寸法が大きすぎる画像（既定: 幅×高さ > 約 3,200 万画素、または長辺 > 10,000px）は**表示しません**。
- 見つけると **`（指定フォルダ）\oversized` へ自動的に移動**します（`delete` と同じく移動であり消去ではありません）。
- これにより巨大画像によるメモリ急増・引っかかりを防ぎます。閾値は実装側で調整できます。
- まれに**移動に失敗し続けた場合**（例：書き込み権限が無い）、最大 5 回試行後に**スライドショーを停止**し、
  対象の**ファイル名を示すダイアログ**を表示します。原本は消えません。ファイルを手動で処理してから、
  「⚙ 設定」でフォルダを選び直すか、アプリを再起動してください。

### 5.5 稼働中に追加した画像の取り込み

- 実行中にフォルダへ画像を追加すると、おおむね 10〜20 秒以内に表示ローテーションへ加わります
  （コピー途中の不完全ファイルを誤読しないよう、サイズ・更新時刻の安定を確認してから取り込みます）。
- 外部で削除・移動した画像は、次回スキャン時に表示順から自然に外れます。

### 5.6 壊れた画像・読み込み失敗

- 開けない画像があってもアプリは落ちません。飛ばして次へ進みます。
- 同じ画像で連続して失敗し続ける場合、そのセッション中は表示対象から外します（次回起動で再試行）。

### 5.7 全画面

- `F11` または背景ダブルクリックで切り替え。マルチモニタではカーソルのあるモニタで全画面になります。

## 6. 管理画面と永続化

### 6.1 管理画面（⚙ 設定）

- **画像フォルダ**: 「参照...」でネイティブのフォルダ選択ダイアログを開きます。
- **切替間隔（秒）**: 5〜60 のスライダー。**即時反映**（確認ダイアログ無し）。
- **表示枚数**: 1 枚 / 2 枚。**即時反映**。2→1 にすると右側が消え、その画像は未表示として後で単独表示されます。

### 6.2 永続化（続きから再開）

- 設定（フォルダ・間隔・枚数）とサイクルの進行（再生順・表示位置・サイクル数）は**ディスクに保存**され、
  アプリを閉じて再起動しても、**同じフォルダなら続きから**再開します。
- 状態ファイルは既定で**実行ファイルと同じフォルダ**の `shufflepic_state.json`（書き込めない場合は
  `%APPDATA%\ShufflePic` などへフォールバック）。
- **フォルダを変更すると、それまでのカウント等はすべてクリア**され、新フォルダで最初から始まります
  （新しい `delete` / `oversized` フォルダも用意します）。

---

## 7. 安全性について

- **原本ファイルは、「削除」または「巨大画像の自動退避」を選んだとき以外、移動・改名・削除されません。**
- 「削除」「退避」は同一ドライブ内での移動で行うため、途中で落ちてもファイルが中途半端になりません。
- 移動は既存の同名ファイルを**上書きしません**（番号付きの別名で退避）。

## 8. 困ったときは（トラブルシューティング）

| 症状 | 対処 |
| --- | --- |
| 起動しても「設定からフォルダを選択してください」のまま | 「⚙ 設定」→「参照...」でフォルダを選ぶ。 |
| 「退避できない画像があるため停止しました」ダイアログ | 表示されたファイル名の画像を手動で処理し、フォルダを選び直すか再起動。 |
| 大きい画像が表示されない | 仕様です。寸法上限超過の画像は `oversized` フォルダへ自動退避されます。 |
| 追加した画像がすぐ出ない | 仕様です。書き込み完了確認のため 10〜20 秒ほど待ってください。 |
| 動作がもたつく | `cargo run --release` で実行しているか確認。 |
| 前回の続きにならない | 別フォルダを選んでいないか確認。フォルダ変更時はリセットされます。 |

## 9. クイックリファレンス

```
起動:        cargo run --release  →（初回）⚙ 設定 でフォルダ選択
再生/停止:   Space  または 上部ボタン  または 右クリックメニュー
全画面:      F11  または 背景ダブルクリック
削除:        画像を右クリック →「削除」（delete フォルダへ退避）
設定:        ⚙ 設定（フォルダ・間隔5〜60秒・表示枚数1/2、即時反映）
続き:        同じフォルダなら次回起動時に自動で続きから
巨大画像:    自動で oversized フォルダへ退避（表示しない）
```
